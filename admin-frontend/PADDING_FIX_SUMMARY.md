# 管理页面内边距修复总结

## 问题描述
所有管理页面（用户管理、模型管理、知识图谱、服务管理）的组件内容紧贴边缘，缺少合适的内边距。

## 修复内容

### 1. 表单容器 (`.form-grid`)
- **修改前**: `padding: 0`
- **修改后**: `padding: 0 22px 20px`
- **影响页面**: 用户管理、模型管理、服务管理、知识图谱的表单区域

### 2. 列表容器 (`.list-stack`, `.simple-list`)
- **修改前**: `padding-right: 4px`
- **修改后**: `padding: 0 22px 20px`
- **影响页面**: 用户管理、模型管理、服务管理的列表区域

### 3. 空状态提示 (`.empty-copy`)
- **修改前**: `padding: 0`
- **修改后**: `padding: 0 22px 20px`
- **影响页面**: 所有页面的空状态提示

### 4. 搜索栏和按钮行 (`.search-row`, `.button-row`)
- **修改前**: `padding: 0`
- **修改后**: `padding: 0 22px` + `.search-row` 添加 `margin-bottom: 16px`
- **影响页面**: 用户管理、模型管理、服务管理的搜索区域

### 5. 表单滚动容器 (`.surface-form`)
- **修改前**: `padding-right: 4px`
- **修改后**: `padding: 0 22px 20px`
- **影响页面**: 所有使用 `.surface-form` 的表单区域

### 6. 知识图谱工具栏 (`.graph-toolbar`)
- **修改前**: `padding-bottom: 12px`
- **修改后**: `padding: 20px 22px 12px`
- **影响页面**: 知识图谱页面的顶部工具栏

### 7. 知识图谱元数据栏 (`.graph-meta-bar`)
- **修改前**: `padding: 8px 0`
- **修改后**: `padding: 8px 22px`
- **影响页面**: 知识图谱页面的节点/关系统计栏

## 设计规范

### 统一的内边距标准
- **左右内边距**: 22px（与 `.card-head` 保持一致）
- **底部内边距**: 20px（为内容提供呼吸空间）
- **顶部内边距**: 
  - 工具栏: 20px
  - 其他内容: 0（由 `.card-head` 的 border-bottom 提供视觉分隔）

### 不需要修改的元素
- `.card-head`: 已有 `padding: 20px 22px 16px`，保持不变
- `.overview-card-scroll`: 已有 `padding: 16px 22px 20px`，保持不变
- `.status-banner`: 已有 `padding: 14px 16px`，保持不变（内部组件有自己的边距）
- `.surface`: 保持 `padding: 0`，由子元素控制内边距
